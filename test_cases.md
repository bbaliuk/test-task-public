# Тест-кейсы для API метода POST /pet/{petId}

## TC-01 Успешное обновление имени и статуса

**Предусловие:**  
Питомец с указанным `petId` существует.

**Шаги:**

1. Отправить POST `/pet/{petId}`
2. Передать formData:
   - `name = "TypicalCatName"`
   - `status = "sold"`

**Ожидаемый результат:**

- HTTP `200`
- Имя и статус питомца обновлены

---

## TC-02 Обновление только имени

**Шаги:**

1. POST `/pet/{petId}`

   - `name = "Murzik"`

**Ожидаемый результат**

- HTTP `200`

- Статус не изменился

---


## TC-02 Обновление только статуса

**Ожидаемый результат**

- Статус изменился

# Тест-кейсы для страницы авторизации Google Sign In

---

## TC-01 Отображение страницы входа

**Шаги**

1. Открыть страницу `https://accounts.google.com/`

**Ожидаемый результат**

- Отображается страница входа
- Видно поле ввода email
- Видна кнопка **Далее**

---

## TC-02 Переход к вводу пароля после ввода email

**Шаги**

1. Открыть страницу входа
2. Ввести корректный email
3. Нажать **Далее**

**Ожидаемый результат**

- Открывается страница ввода пароля
- Отображается поле для ввода пароля

---

## TC-03 Успешная авторизация пользователя

**Шаги**

1. Открыть страницу входа
2. Ввести корректный email
3. Нажать **Далее**
4. Ввести корректный пароль
5. Нажать **Далее**

**Ожидаемый результат**

- Происходит вход в аккаунт
- Пользователь перенаправляется на `https://myaccount.google.com`
- Отображается имя пользователя и email

---

## TC-04 Попытка входа без ввода email

**Шаги**

1. Открыть страницу входа
2. Не вводить email
3. Нажать **Далее**

**Ожидаемый результат**

- Отображается сообщение об ошибке  
  `Введите адрес электронной почты или номер телефона.`

---

## TC-05 Проверка валидации некорректного email

**Шаги**

1. Открыть страницу входа
2. Ввести `test`
3. Нажать **Далее**

**Ожидаемый результат**

- Отображается сообщение о некорректном формате email

---

## TC-06 Попытка входа с неправильным паролем

**Шаги**

1. Открыть страницу входа
2. Ввести корректный email
3. Нажать **Далее**
4. Ввести неверный пароль
5. Нажать **Далее**

**Ожидаемый результат**

- Отображается сообщение об ошибке
- Вход в аккаунт не происходит

---

## TC-07 Повторная попытка входа после ошибки

**Шаги**

1. Ввести неверные данные для входа
2. Получить сообщение об ошибке
3. Нажать кнопку **Повторить попытку**

**Ожидаемый результат**

- Пользователь возвращается к форме входа

---

## TC-08 Переключение языка интерфейса

**Шаги**

1. Открыть страницу входа
2. Открыть список выбора языка
3. Выбрать **English**

**Ожидаемый результат**

- Интерфейс страницы меняется на английский язык
- Заголовок страницы отображается как `Sign in`

---

## TC-09 Проверка отображения основных элементов страницы

**Шаги**

1. Открыть страницу входа

**Ожидаемый результат**

Отображаются элементы:

- поле ввода email
- кнопка **Далее**
- переключатель языка
- текст приветствия

---

## TC-10 Проверка отображения поля пароля

**Шаги**

1. Открыть страницу входа
2. Ввести корректный email
3. Нажать **Далее**

**Ожидаемый результат**

- Отображается поле ввода пароля
---

## TC-11 Проверка повторного ввода email после ошибки

**Шаги**

1. Открыть страницу входа
2. Нажать **Далее** без ввода email
3. Получить сообщение об ошибке
4. Ввести корректный email

**Ожидаемый результат**

- Сообщение об ошибке исчезает
- Пользователь может продолжить авторизацию
