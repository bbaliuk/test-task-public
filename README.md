# Test Automation Project

Автоматизированные тесты для:

- API метода `POST /pet/{petId}` из проекта https://petstore.swagger.io/
- UI страницы авторизации Google

---
## Установка

1. Клонировать репозиторий:

```bash
git clone <repo_url>
````

2. Перейти в директорию проекта:

```bash
cd project
```

3. Создать виртуальное окружение:

```bash
python -m venv venv
```

4. Активировать виртуальное окружение:

**Windows:**

```bash
venv\Scripts\activate
```

**Linux / Mac:**

```bash
source venv/bin/activate
```

5. Установить зависимости:

```bash
pip install -r requirements.txt
```

## Запуск тестов

Запуск всех тестов:

```bash
pytest
```
## Паттерны

В проекте используются:

- Page Object Model
- Page element
- Pydantic модели для API


## Реализованные тесты

### API

Тесты для метода: ```POST /pet/{petId}```

Проверяются сценарии:

- обновление имени питомца
- обновление статуса
- обновление имени и статуса
- обработка несуществующего ID
- 
### UI

Тестирование страницы авторизации Google:

Проверяются сценарии:

- вход без email
- переключение языка
- ввод email и пароля

---
Однако стоит подчеркнуть, что Google против автоматизации тестирования их страниц, 
вследствие чего, с запуском UI тестов могут возникнуть трудности

---

Так же, в процессе тестирования API было найдено несколько багов, один из которых критический. А именно то, 
что само API может создавать petId за пределами int64, которые в последствии не могут быть обработаны.
Так или иначе в нем не подразумевается никакой валидации никаких значений, оно радостно отдает 200 на все, что можно "засунуть"
в запрос   

## Использование нейросетей

При непосредственном написании тестов, кейсов и фреймворка нейросети намеренно не использовались. Однако,
они использовались для написания локаторов UI тестов, форматирования тест-кейсов для улучшения читабельности, 
в некоторых случаях для pydantic моделей, и, отчасти, этого README файла.
